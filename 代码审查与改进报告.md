# 进程调度算法模拟系统 —— 代码审查与改进报告

> **审查日期**: 2026年3月9日  
> **项目名称**: 进程调度算法的设计与实现（操作系统课程设计）  
> **开发语言**: C++11  
> **源文件数量**: 11 个（1 个 `.cpp` + 9 个 `.h` + 1 个 `build.bat`）

---

## 一、项目结构概览

```
src/
├── main.cpp               # 主程序（菜单系统、用户交互）
├── PCB.h                  # 进程控制块定义
├── ProcessQueue.h         # 就绪队列管理
├── Scheduler.h            # 调度算法抽象基类
├── FCFS_Scheduler.h       # 先来先服务调度
├── SJF_Scheduler.h        # 短作业优先 / SRTF 调度
├── Priority_Scheduler.h   # 优先级调度（非抢占 / 抢占 / 老化）
├── RR_Scheduler.h         # 时间片轮转调度
├── HRRN_Scheduler.h       # 高响应比优先调度
├── MLFQ_Scheduler.h       # 多级反馈队列调度
├── AlgorithmComparator.h  # 多算法对比分析模块
├── ProcessManager.h       # 进程管理模块
└── build.bat              # 编译脚本
```

**设计特点**: 采用面向对象设计，`Scheduler` 为抽象基类定义统一接口，各调度算法通过继承实现 `schedule()` 虚函数。所有进程操作通过 `ProcessManager` 统一管理，`AlgorithmComparator` 提供横向对比功能。

## 二、审查发现的问题汇总

共发现 **11 个问题**，按严重程度分类如下：

### 2.1 致命缺陷（会导致程序崩溃）

#### 问题 #1: `printResults()` 标题行填充长度下溢导致崩溃

| 项目 | 内容 |
|------|------|
| **文件** | `Scheduler.h` 第 107 行 |
| **严重度** | ★★★★★ 致命 |
| **症状** | MLFQ 等长名称算法调用 `printResults()` 时程序崩溃 |

**问题代码:**
```cpp
std::string(60 - algorithmName.length() * 1.5, ' ')
```

**根因分析:**  
`algorithmName` 含中文字符（UTF-8 编码每个中文占 3 字节），`std::string::length()` 返回的是**字节数**而非显示宽度。例如 MLFQ 的算法名 `"多级反馈队列(MLFQ, 3级, q=1/2/4)"` 的 `length()` 约为 48 字节，`48 × 1.5 = 72 > 60`，计算结果下溢为极大的 `size_t` 值（无符号整数回绕），导致试图分配数 GB 内存而崩溃。

#### 问题 #2: `generateRandomProcesses()` 中 `std::stoi` 无异常保护

| 项目 | 内容 |
|------|------|
| **文件** | `ProcessManager.h` 第 126-131 行 |
| **严重度** | ★★★★★ 致命 |
| **症状** | 用户输入非数字字符串时程序终止 |

**问题代码:**
```cpp
maxArrival = input.empty() ? 20 : std::stoi(input);  // "abc" → 抛出异常
maxBurst   = input.empty() ? 15 : std::stoi(input);
maxPriority = input.empty() ? 10 : std::stoi(input);
```

**根因分析:**  
`std::stoi` 在参数不可解析为整数时抛出 `std::invalid_argument` 异常，程序未捕获该异常，导致直接终止。用户随意输入即可触发。

### 2.3 显示缺陷

#### 问题 #4: 中文字符在表格和 Gantt 图中对齐错位

|------|------|

| **严重度** | ★★★ 中 |

## 三、改进措施与实施

### 3.1 修复 #1: 标题行填充安全计算

**修改文件:** `Scheduler.h`

新增 `displayWidth()` 静态方法，通过分析 UTF-8 字节头判断字符宽度（ASCII 占 1 列，CJK 占 2 列）：

```cpp
static int displayWidth(const std::string& s) {
    int width = 0;
    for (size_t i = 0; i < s.size(); ) {
        unsigned char c = s[i];
        if (c < 0x80) { width += 1; i += 1; }
        else if (c < 0xE0) { width += 1; i += 2; }
        else if (c < 0xF0) { width += 2; i += 3; }
        else { width += 2; i += 4; }
    }
    return width;
}
```

标题行改为安全计算：
```cpp
int titlePad = std::max(1, 72 - displayWidth(algorithmName));
```

同时修复了 Gantt 图中"空闲"标签的宽度计算（问题 #4 同步解决）。

---

### 3.2 修复 #2: 安全的字符串转整数

**修改文件:** `ProcessManager.h`

新增 `safeStoi()` 私有静态方法，用 try-catch 包裹 `std::stoi`：

```cpp
static int safeStoi(const std::string& s, int defaultVal) {
    if (s.empty()) return defaultVal;
    try { return std::stoi(s); }
    catch (...) { return defaultVal; }
}
```

三处 `std::stoi` 调用全部替换为 `safeStoi`，非法输入回退到默认值而非崩溃。

---
