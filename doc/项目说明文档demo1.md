## 一、项目概述

本项目是操作系统课程设计作品，题目为 **"进程调度算法的设计与实现"**。系统采用 C++11 面向对象编程，通过继承与多态设计，实现了 7 种经典 CPU 调度算法（共 8 种运行模式），并提供了完整的进程管理、算法运行、结果可视化、多算法横向比较等功能。

**系统特点：**
- 实现 7 种调度算法：FCFS、SJF、SRTF、优先级调度（非抢占/抢占+老化）、RR、HRRN、MLFQ
- 面向对象架构：抽象基类 + 具体算法子类，易于扩展
- 全中文交互界面：精美的 Unicode 边框菜单
- 丰富的可视化：Gantt 图、柱状图、对比表格
- 完善的进程管理：手动输入、批量添加、随机生成、预设用例、文件导入导出
- 多算法横向比较与智能推荐

## 四、核心数据结构

### 4.1 进程控制块 PCB

**文件：** `PCB.h`

PCB（Process Control Block）是操作系统中描述进程的核心数据结构。本系统的 PCB 包含以下属性：

**静态属性（用户输入）：**

| 字段名       | 类型        | 说明                                  |
|-------------|-------------|---------------------------------------|
| `pid`       | `int`       | 进程 ID，系统自动分配                  |
| `name`      | `string`    | 进程名称（如 P1, P2）                 |
| `arrivalTime` | `int`     | 到达时间（进程进入系统的时刻）          |
| `burstTime` | `int`       | CPU 区间时间 / 服务时间                |
| `priority`  | `int`       | 优先级（数值越小优先级越高）            |

**动态属性（调度过程中自动计算）：**

| 字段名              | 类型        | 说明                                      |
|---------------------|-------------|-------------------------------------------|
| `remainingTime`     | `int`       | 剩余服务时间（抢占式算法使用）              |
| `startTime`         | `int`       | 首次获得 CPU 运行的时刻                    |
| `completionTime`    | `int`       | 进程完成的时刻                             |
| `waitingTime`       | `int`       | 等待时间 = 周转时间 − 服务时间             |
| `turnaroundTime`    | `int`       | 周转时间 = 完成时间 − 到达时间             |
| `weightedTurnaround`| `double`    | 带权周转时间 = 周转时间 / 服务时间         |
| `responseTime`      | `int`       | 响应时间 = 首次运行时间 − 到达时间         |
| `state`             | `ProcessState` | 当前进程状态                            |
| `hasStarted`        | `bool`      | 是否已开始执行                             |
| `currentQueue`      | `int`       | 当前所在队列级别（MLFQ 专用）              |

**进程状态枚举 ProcessState：**

| 枚举值        | 含义   | 说明                       |
|--------------|--------|----------------------------|
| `NEW`        | 新建   | 进程已创建但尚未到达        |
| `READY`      | 就绪   | 进程在就绪队列中等待 CPU    |
| `RUNNING`    | 运行   | 进程正在 CPU 上执行         |
| `WAITING`    | 等待   | 进程等待 I/O 或其他资源     |
| `TERMINATED` | 终止   | 进程已执行完成              |

**关键方法：**
- `calculateMetrics()`：根据完成时间计算周转时间、等待时间、带权周转时间、响应时间
- `reset()`：重置所有动态属性为初始值（切换算法前调用）
- `getStateStr()`：返回进程状态的中文字符串

### 4.2 调度事件 ScheduleEvent

**文件：** `PCB.h`

用于记录 Gantt 图中的每一个调度事件（时间片）：

| 字段名        | 类型     | 说明                               |
|--------------|----------|------------------------------------|
| `processName`| `string` | 执行的进程名称（"IDLE" 表示空闲）    |
| `startTime`  | `int`    | 事件开始时间                        |
| `endTime`    | `int`    | 事件结束时间                        |

### 4.3 进程就绪队列 ProcessQueue

**文件：** `ProcessQueue.h`

封装了就绪队列的常用操作，内部使用 `vector<PCB*>` 存储进程指针（避免拷贝开销）。

**基础操作：**

| 方法             | 说明                               |
|------------------|------------------------------------|
| `enqueue(PCB*)`  | 将进程加入队尾                      |
| `dequeue()`      | 从队头取出进程并返回                 |
| `peek()`         | 查看队头进程（不移出）               |
| `remove(PCB*)`   | 移除指定进程                        |
| `empty()`        | 判空                               |
| `size()`         | 返回队列大小                        |
| `clear()`        | 清空队列                           |

**排序操作（为不同调度算法服务）：**

| 方法                     | 说明                               |
|--------------------------|------------------------------------|
| `sortByArrivalTime()`    | 按到达时间升序排列（FCFS 使用）       |
| `sortByBurstTime()`      | 按服务时间升序排列（SJF 使用）        |
| `sortByRemainingTime()`  | 按剩余时间升序排列（SRTF 使用）       |
| `sortByPriority()`       | 按优先级排序（数值小优先，优先级调度用）|
| `sortBy(comparator)`     | 通用自定义排序，接受 lambda          |

**查找操作：**

| 方法                       | 说明                                     |
|----------------------------|------------------------------------------|
| `findShortestRemaining()`  | 查找剩余时间最短的进程（SRTF 使用）        |
| `findHighestPriority()`    | 查找最高优先级的进程（优先级调度使用）      |

> 所有排序均以 `arrivalTime` 或 `pid` 作为第二排序键，保证同值时调度的确定性。

### 4.4 算法性能指标 AlgorithmMetrics

**文件：** `AlgorithmComparator.h`

用于存储一种算法的综合性能指标，方便多算法横向比较：

| 字段名                  | 类型     | 说明            |
|------------------------|----------|-----------------|
| `name`                 | `string` | 算法名称         |
| `avgTurnaroundTime`    | `double` | 平均周转时间     |
| `avgWeightedTurnaround`| `double` | 平均带权周转时间 |
| `avgWaitingTime`       | `double` | 平均等待时间     |
| `avgResponseTime`      | `double` | 平均响应时间     |
| `cpuUtilization`       | `double` | CPU 利用率 (%)   |
| `throughput`           | `double` | 吞吐量           |

## 五、调度算法详解

### 5.1 先来先服务 FCFS

**文件：** `FCFS_Scheduler.h` （89 行）

**算法原理：**
按进程到达就绪队列的先后顺序分配 CPU。先到达的进程先执行，直到该进程运行结束才调度下一个。

**特性：**
- 类型：**非抢占式**
- 优点：实现简单、公平
- 缺点：平均等待时间长、存在"护航效应"（短作业被长作业阻塞）
- 适用场景：批处理系统

**实现流程：**
1. 将所有进程按到达时间排序
2. 按顺序逐个处理：如果当前时间早于进程到达时间，则 CPU 空闲等待（记录 IDLE 事件）
3. 执行进程直至完成，记录开始时间、完成时间
4. 计算各项性能指标

**时间复杂度：** $O(n \log n)$（排序） + $O(n)$（单遍扫描）= $O(n \log n)$

---

### 5.2 短作业优先 SJF（非抢占）

**文件：** `SJF_Scheduler.h` （183 行，`preemptive = false` 模式）

**算法原理：**
在所有已到达的就绪进程中，选择服务时间最短的进程优先执行。执行期间不可抢占。

**特性：**
- 类型：**非抢占式**
- 优点：可以证明在非抢占条件下，平均等待时间最小（最优）
- 缺点：需要预知进程服务时间，长作业可能饥饿
- 适用场景：批处理系统

**实现流程：**
1. 时钟循环推进，每个时刻检查新到达的进程加入就绪队列
2. 就绪队列按服务时间升序排列
3. 取出队首进程（最短者）运行至完成
4. 若就绪队列为空但仍有未完成进程，CPU 空闲等待下一个到达

**时间复杂度：** $O(n^2)$（每次调度需排序或查找最短进程）

---

### 5.3 最短剩余时间优先 SRTF（抢占）

**文件：** `SJF_Scheduler.h` （183 行，`preemptive = true` 模式）

**算法原理：**
SJF 的抢占版本。每当有新进程到达时，若新进程的服务时间比当前正在运行进程的剩余时间更短，则抢占当前进程。

**特性：**
- 类型：**抢占式**
- 优点：平均等待时间比非抢占 SJF 更优
- 缺点：频繁上下文切换，长作业饥饿更严重
- 适用场景：对响应时间要求较高的系统

**实现流程：**
1. 以事件驱动方式推进（时钟逐单位递增）
2. 每个时刻：接纳新到达进程 → 在就绪队列中找剩余时间最短者 → 执行 1 个时间单位
3. 若有更短进程到达，自动抢占（即下个时刻重新选择最短者）
4. 利用 `recordEvent()` 的合并逻辑，连续执行同一进程的多个时间单位自动合并为一个事件

**关键优化：** 当就绪队列为空但有未完成进程时，时钟直接跳转到下一个到达时间，避免无效循环。

**时间复杂度：** $O(n \cdot T)$，其中 $T$ 为总调度时长

---

### 5.4 优先级调度（非抢占/抢占 + 老化机制）

**文件：** `Priority_Scheduler.h` （207 行）

**算法原理：**
为每个进程赋予优先级，调度时选择优先级最高（数值最小）的进程执行。支持两种模式以及可选的老化机制。

**模式说明：**

| 模式       | 行为                                                |
|-----------|-----------------------------------------------------|
| 非抢占式   | 选中最高优先级进程后运行至完成，期间不被打断            |
| 抢占式     | 每个时间单位检查，若有更高优先级进程到达则抢占           |

**老化机制（Aging）：**
- 目的：防止低优先级进程饥饿
- 方法：等待中的进程每经过一定时间，优先级数值减 1（即优先级提升）
- 下限：优先级最低为 0（最高优先级）
- 在本实现中，每个时间单位都对所有等待中的进程执行一次老化检查

**时间复杂度：**
- 非抢占：$O(n^2)$
- 抢占：$O(n \cdot T)$

---

### 5.5 时间片轮转 RR

**文件：** `RR_Scheduler.h` （122 行）

**算法原理：**
所有就绪进程按 FIFO 排列。每个进程分配一个时间片（time quantum），在时间片内执行。若时间片用完进程未结束，则该进程放回队尾等待下一轮。

**特性：**
- 类型：**抢占式**
- 时间片可配置（1~20）
- 优点：公平，响应时间好
- 缺点：时间片过大退化为 FCFS，过小导致上下文切换开销大
- 适用场景：分时系统

**实现流程：**
1. 将已到达的进程按到达时间排入就绪队列
2. 取出队首进程，执行 min(timeQuantum, remainingTime) 个时间单位
3. 在执行期间检查新到达的进程，先于被中断进程入队
4. 若进程未完成，放回队尾
5. 重复直到所有进程完成

**关键细节：** 在时间片执行期间新到达的进程先入队（join before rotation），保证新进程在被轮转进程之前排队。

**时间复杂度：** $O(n \cdot T / q)$，其中 $q$ 为时间片大小

**时间片影响分析：** 系统提供 `analyzeTimeQuantumEffect()` 功能，以 q = {1, 2, 3, 4, 5, 8, 10, 15, 20} 运行 RR 算法并对比各指标变化趋势。

---

### 5.6 高响应比优先 HRRN

**文件：** `HRRN_Scheduler.h` （108 行）

**算法原理：**
结合了 FCFS 和 SJF 的优点。每次调度时计算所有就绪进程的响应比，选择响应比最高的进程执行。

**响应比公式：**

$$R = \frac{W + S}{S} = 1 + \frac{W}{S}$$

其中：
- $W$ = 等待时间 = 当前时间 − 到达时间
- $S$ = 服务时间（burstTime）

**特性：**
- 类型：**非抢占式**
- 优点：短作业天然响应比高（服务时间小），长作业等待越久响应比也越高，兼顾二者
- 缺点：每次调度需计算所有进程的响应比，开销略大
- 适用场景：批处理系统
- **不会产生饥饿**

**实现流程：**
1. 检查所有已到达的就绪进程
2. 计算每个进程的响应比
3. 选择响应比最高的进程执行至完成
4. 重复直到所有进程完成

**时间复杂度：** $O(n^2)$

---

### 5.7 多级反馈队列 MLFQ

**文件：** `MLFQ_Scheduler.h` （195 行）

**算法原理：**
设置多个优先级不同的就绪队列（级数可配置，默认 3 级）。每级队列分配不同的时间片（通常逐级递增：2^i）。新进程首先进入最高级队列；如果进程在当前级别的时间片内未完成，则降级到下一级队列；高级队列不空时不调度低级队列。

**参数配置：**

| 参数            | 默认值      | 说明                              |
|----------------|-------------|-----------------------------------|
| 队列级数        | 3           | 可设 2~5                          |
| 各级时间片      | {1, 2, 4}   | 通过 `configure()` 自定义         |
| 优先级提升      | 关闭        | 可选开启，设置提升间隔（如 20）    |

**特性：**
- 类型：**抢占式**
- 优点：综合性能最好，自动适应不同类型进程（短作业在高级队列快速完成，长作业逐步降级）
- 缺点：实现复杂，参数调优困难
- 适用场景：通用操作系统

**实现流程：**
1. 新进程到达时进入第 0 级（最高优先级）队列
2. 始终从最高级非空队列取出进程执行
3. 若进程在当前级时间片内完成，则终止
4. 若时间片用完未完成，进程降级到下一级队列（最低级不再降级）
5. 若有新进程到达高级队列，抢占当前低级队列的进程
6. 若启用优先级提升：每隔指定间隔将所有进程提升到第 0 级队列

**防饥饿机制 (Priority Boost)：**
定期（如每 20 个时间单位）将所有等待中的进程重新提升到最高级队列，确保低级队列的进程不会永远得不到执行。

**时间复杂度：** $O(n \cdot T)$

## 十二、设计特色与创新点

- **Gantt 图**：文本形式的时间线图，宽度与时长成正比
- **柱状图**：控制台文本柱状图（使用 Unicode 块字符 `█` `▓`）
- **格式化表格**：Unicode 边框（`╔ ╗ ╠ ╣ ╚ ╝`）的美观表格

### 12.4 多算法横向对比与智能推荐

不仅能单独运行某一算法，还能同时运行多种或全部算法，自动：
- 生成 6 项指标的对比表

- 给出不同应用场景的具体推荐

### 12.5 完善的进程管理

支持 5 种进程创建方式（手动、批量、随机、预设、文件导入），以及编辑/删除/导出等完整 CRUD 操作。内置 5 组覆盖不同场景的预设测试用例。

## 十三、性能评价指标公式

| 指标           | 公式                                                     | 含义说明                              |
|---------------|----------------------------------------------------------|---------------------------------------|
| 周转时间       | $T_t = T_{完成} - T_{到达}$                                | 进程从提交到完成的总时间                |
| 带权周转时间   | $T_w = \frac{T_t}{T_{服务}}$                               | 周转时间与服务时间的比值，越接近 1 越好  |
| 等待时间       | $T_{wait} = T_t - T_{服务}$                                | 进程在就绪队列中等待的总时间            |
| 响应时间       | $T_r = T_{首次运行} - T_{到达}$                             | 进程从提交到首次获得 CPU 的时间         |
| CPU 利用率     | $U = \frac{T_{busy}}{T_{total}} \times 100\%$             | CPU 忙碌时间占总时间的百分比            |
| 吞吐量         | $\lambda = \frac{n}{T_{total}}$                           | 单位时间内完成的进程数                  |

其中：
- 平均周转时间：$\bar{T_t} = \frac{1}{n}\sum_{i=1}^{n} T_{t_i}$
- 平均带权周转时间：$\bar{T_w} = \frac{1}{n}\sum_{i=1}^{n} T_{w_i}$
- 平均等待时间：$\bar{T_{wait}} = \frac{1}{n}\sum_{i=1}^{n} T_{wait_i}$
- 平均响应时间：$\bar{T_r} = \frac{1}{n}\sum_{i=1}^{n} T_{r_i}$

## 十四、算法时间复杂度分析

设 $n$ 为进程数，$T$ 为调度总时长，$q$ 为时间片大小。

| 算法                | 时间复杂度     | 空间复杂度  | 说明                              |
|--------------------|---------------|------------|-----------------------------------|
| FCFS               | $O(n \log n)$ | $O(n)$     | 排序一次 + 线性扫描               |
| SJF（非抢占）       | $O(n^2)$      | $O(n)$     | 每次从就绪队列选最短，共 n 次      |
| SRTF（抢占）        | $O(n \cdot T)$| $O(n)$     | 每个时间单位检查就绪队列           |
| 优先级（非抢占）     | $O(n^2)$      | $O(n)$     | 每次选最高优先级，共 n 次          |
| 优先级（抢占）       | $O(n \cdot T)$| $O(n)$     | 每个时间单位检查优先级             |
| RR                 | $O(n \cdot T/q)$ | $O(n)$  | 每个时间片调度一次                 |
| HRRN               | $O(n^2)$      | $O(n)$     | 每次计算所有响应比并选最大          |
| MLFQ               | $O(n \cdot T)$| $O(n \cdot k)$ | k 为队列级数，每时刻检查各级队列 |
